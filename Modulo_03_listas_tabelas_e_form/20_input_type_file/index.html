Input type="file"

Agora iremos ver um campo para enviar arquivos, para podermos enviar arquivos junto de nosso formulario. Geralemente arquivos sao enviados atraves do method="POST".

Para adicionarmos arquivos iremos usar o type="file", com o usuario conseguira enviar arquivos. Com ele podemos tambem, especificar os tipos de arquivos que poderam ser enviados pelo usuario. Para isso iremos adicionar o atributo chamado accept="", nele temos as opcoes:

  • input="title" accept="image/*": com ele sera aceito qualquer tipo de imagem.
  • input="title" accept="aplication/pdf": para aceitar pdf

Obs: quando temos um arquivo type="file", temos que adicionar uma outra propriedade em nosso form, o nome da propriedade e enctype para definir como os dados do formulário serão codificados (encodados) quando forem enviados ao servidor.

Usos comuns do enctype:

application/x-www-form-urlencoded (padrão):

  • É o modo padrão.
  • Os dados são codificados como pares chave=valor e enviados na URL (em POST, no corpo).
  • Ex: nome=David&idade=25.

multipart/form-data:

  • Necessário quando você vai enviar arquivos (< input type="file" >).
  • Os dados são divididos em "partes", com os arquivos incluídos corretamente.
  • Cada parte pode ter um tipo de dado diferente (texto, arquivo etc).
  • Este é o que você precisa usar para uploads.

text/plain:

  • Raramente usado.
  • Envia os dados como texto simples, separados por quebras de linha.
  • Não recomendado para formulários reais (não suporta arquivos nem codifica os dados corretamente).
