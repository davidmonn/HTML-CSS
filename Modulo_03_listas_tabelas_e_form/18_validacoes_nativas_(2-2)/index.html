Validacao Nativa (2/2)

Sempre que temos um formulario, quando clicamos no botao de "envio" do form, geralmente a gente faz um processo antes de enviar, chamado de validacao esse processo ira verificar basicamente se os dados estao preenchidos e se os dados que estao preenchidos, estao de acordo com o minimo que precisamos, Ex: um campo de idade, nao faria sentido ter letras, apenas numeros e o suficiente.

Iremos ver os principais tipos de pre validacoes nativas:

  • required: torna o campo obrigatorio.
  • minlength: minimo de caracteres que o usuario precisara para preencher o campo.
  • maxlength: maximo de caracteres que o usuario podera usar para preencher o campo.
  • step: em um type com number, podemos usar step para quando o usuario clicar em escolher idade ele ira pulando para quantidade que tiver no step.
  • min: minimo que o usuario tera que ter para acessar o site.
  • max: maximo que o usuario podera ter, como usamos idade, entao o maximo que o usuario podera ter de idade.
